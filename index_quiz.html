Знайомство з фронтенд

1 Які основні завдання фронтенд-розробника на етапі дизайну веб-додатка?

  1. Замовлення графічного обладнання.
  2. Створення графічного дизайну та розробка графічних елементів інтерфейсу.
  3. Оптимізація бази даних.
  4. Розробка серверної частини.

2 Які технології використовуються в процесі фронтенд-розробки для надання візуального оформлення веб-додатка?

  1. Python.
  2. HTML, CSS, JavaScript.
  3. SQL.
  4. Java.

3 Чому адаптивний дизайн є важливим аспектом на етапі розробки веб-додатка?

  1. Для створення креативних анімацій.
  2. Для покращення безпеки бази даних.
  3. Для забезпечення коректного відображення на різних пристроях та екранах.
  4. Для розробки штучного інтелекту.

4 Які основні завдання включає в себе етап "Інтеграція та тестування" у процесі розробки веб-додатка?

  1. Створення серверної частини додатка.
  2. Оптимізація продуктивності та швидкості завантаження.
  3. Інтеграція фронтенду та бекенду, тестування на різних етапах, виявлення та виправлення помилок.
  4. Розробка графічного дизайну.

5 Що таке інтернет і як він працює?

  1. Інтернет – це мережа комп’ютерів, яка працює за допомогою телефонної системи.
  2. Інтернет – це глобальна мережа комп’ютерів, що обмінюються конвертами з повідомленнями.
  3. Інтернет – це мережа комп’ютерів, яка працює за допомогою TCP/IP і дозволяє обмін даними між ними з великою швидкістю.
  4. Інтернет – це спеціальна мережа для обміну мультимедійним контентом.

6 Що характеризує імперативні мови програмування?

  1. Програміст визначає бажаний результат, а виконавець обирає, як його досягти.
  2. Програміст вказує комп'ютеру кроки, які потрібно виконати для досягнення бажаного результату.
  3. Програма представляє собою послідовність інструкцій, які виконуються в певному порядку.
  4. Методологія розробки програм, спрямована на оптимізацію виконання комп'ютерних програм.

7 Яка мова програмування є прикладом декларативної мови?

  1. Python.
  2. SQL.
  3. Java.
  4. JavaScript.

8 Яку роль відіграє протокол HTTP у веб-розробці та яка основна мета його використання?

  1. HTTP дозволяє обмінюватися даними між веб-серверами та веб-клієнтами.
  2. HTTP використовується для створення структури веб-сторінок.
  3. HTTP забезпечує конфіденційність та цілісність даних під час їх передачі.
  4. HTTP визначає стилі та оформлення веб-сайтів.

9 Яка основна відмінність між HTTP і HTTPS?

  1. HTTPS використовується для безпечної передачі даних та використовує шифрування.
  2. HTTP використовується для безпечної передачі даних.
  3. HTTP та HTTPS ідентичні за своєю суттю.
  4. HTTP використовується для статичних веб-сайтів, а HTTPS - для динамічних.

10 Яким чином мова HTML впливає на структуру веб-сторінок?

  1. HTML визначає кольори, шрифти та інші стилізаційні параметри веб-сторінок.
  2. HTML використовується для додавання інтерактивності та функціональності на веб-сторінках.
  3. HTML є мовою розмітки, яка визначає структуру та контент веб-сторінок.
  4. HTML контролює виконання скриптів на сторінці та взаємодіє з базою даних.

Знайомство із HTML

1 Що представляють собою теги в мові HTML і яке їх призначення?

  1. Теги визначають атрибути форматування документа.
  2. Теги - це зарезервовані послідовності символів, які вказують браузеру виконувати певні завдання, такі як створення абзацу чи вставка зображення.
  3. Теги використовуються для визначення математичних рівнянь у HTML-документах.
  4. Теги є способом оформлення коментарів у HTML-коді.

2 У чому полягає відмінність між парними тегами та непарними тегами в HTML?

  1. Парні теги використовуються для визначення текстових елементів, а непарні - для визначення зображень та інших мультимедійних об'єктів.
  2. Парні теги використовуються для форматування тексту, а непарні - для форматування зображень.
  3. Парні теги складаються з відкриваючого та закриваючого елементів і визначають блоки контенту, тоді як непарні теги використовуються самостійно.
  4. Парні теги визначають метатеги документа, а непарні - структурні елементи сторінки.

3 Яким чином визначається заголовок сторінки в HTML-документі, і де він розташовується в коді HTML?

  1. Заголовок задається тегом < header>, розташованим всередині тега < body>.
  2. Заголовок задається тегом < title>, розташованим всередині тега < body>.
  3. Заголовок задається тегом < h1> і розташовується всередині тега < head>.
  4. Заголовок задається тегом < h2> і розташовується всередині тега < header>.

4 Для чого використовується тег < meta> в HTML-документі, і які атрибути можуть бути використані в цьому тегу?

  1. Тег < meta> використовується для відображення зображень на сторінці, а атрибути - image і source.
  2. Тег < meta> використовується для задання ключових слів та опису сторінки для пошукових систем, а атрибути - keywords і description.
  3. Тег < meta> використовується для створення таблиць на сторінці, а атрибути - table і columns.
  4. Тег < meta> використовується для визначення мови веб-сторінки, а атрибут - lang.

5 Яка різниця між ненумерованим (< ul>), нумерованим (< ol>) і описовим (< dl>) списками в HTML, і як виглядає приклад кожного типу списку в коді HTML?

  1. Ненумерований список використовується для нумерації пунктів, нумерований - для відображення ненумерованих пунктів, а описовий - для створення пар слів і визначень.
  2. Ненумерований список використовується для нумерації пунктів, нумерований - для відображення нумерованих пунктів, а описовий - для маркування термінів і їх визначень.
  3. Ненумерований список використовується для відображення ненумерованих пунктів, нумерований - для нумерації пунктів, а описовий - для визначення ключових слів на сторінці.
  4. Ненумерований список використовується для відображення ненумерованих пунктів, нумерований - для відображення нумерованих пунктів, а описовий - для створення пар слів і визначень.

6 Які є відмінності між абсолютним та відносним шляхами в контексті веб-розробки?

  1. Абсолютний шлях завжди починається з кореневого каталогу, тоді як відносний шлях вказує місце відносно поточного робочого каталогу.
  2. Абсолютний шлях використовується для локальних ресурсів, а відносний - для зовнішніх ресурсів.
  3. Абсолютний шлях вказує на розташування файлу в Інтернеті, тоді як відносний шлях - в локальній файловій системі.
  4. Абсолютний шлях використовується для відносних посилань, а відносний - для абсолютних посилань.

7 Які поради з форматування коду HTML наведено в тексті, і чому вони важливі для розробників?

  1. Використовуйте відступи для вирівнювання елементів, використовуйте тільки одинарні лапки для атрибутів, вирівнюйте атрибути, використовуйте нижній регістр для тегів та атрибутів.
  2. Не використовуйте відступи, використовуйте лише подвійні лапки для атрибутів, не вирівнюйте атрибути, використовуйте будь-який регістр для тегів та атрибутів.
  3. Використовуйте табуляцію для вирівнювання елементів, використовуйте тільки одинарні лапки для атрибутів, не вирівнюйте атрибути, використовуйте нижній регістр для тегів та атрибутів.
  4. Використовуйте відступи для вирівнювання елементів, використовуйте тільки подвійні лапки для атрибутів, вирівнюйте атрибути, використовуйте будь-який регістр для тегів та атрибутів.

9 Які вбудовані стилі елементу для стилізації тексту в HTML наведено в тексті? Наведіть приклад використання одного із цих стилів.

Git. GitHub.

1 Що робить система контролю версій (VCS)?

  1. Записує зміни у файл або набір файлів протягом часу
  2. Створює нові файли
  3. Видаляє файли
  4. Запускає веб-сервер

2 Яка основна відмінність між централізованими та розподіленими системами контролю версій (СКВ)?

  1. Централізовані СКВ мають єдиний сервер, тоді як розподілені СКВ дозволяють копіювати репозиторій повністю.
  2. Розподілені СКВ взагалі не використовують сервери.
  3. Централізовані СКВ не використовують сервери.
  4. Розподілені СКВ не зберігають історію змін файлів.

3 Як Git зберігає дані, в чому основна відмінність від інших систем керування версіями?

  1. У вигляді списку змін у файлах
  2. У вигляді набору знімків мініатюрної файлової системи
  3. У вигляді текстового опису змін
  4. У вигляді таблиці зі змінами

4 Що робить команда "коміт" в Git?

  1. Видаляє файли
  2. Зберігає стан проекту та зміни в базі Git
  3. Переміщує файли до каталогу Git
  4. Зберігає тільки зміни в окремому файлі

5 Що має допомагати повідомлення комітів за імперативним стилем дій?

  1. Пояснити найважливіші факти у якнайкоротшій формі
  2. Збільшити кількість слів у повідомленні
  3. Використовувати складні терміни та фрази
  4. Описувати всі можливі зміни в проекті

6 Як перевірити наявність Git на локальній машині за допомогою командного рядка в Windows?

  1. git version
  2. git check
  3. git verify
  4. git present

7 Якщо Git вже встановлений, але його версія застаріла, яку команду можна використати для оновлення?

  1. git upgrade
  2. git update
  3. git update-git-for-windows
  4. git update-version

8 Яка роль Octocat у веб-сервісі GitHub?

  1. Мессенджер для комунікації між розробниками.
  2. Технічний підтримка GitHub.
  3. Талісман сервісу GitHub.
  4. Інтерфейс для створення графічних дизайнів.

9 Що надає GitHub розробникам для спільної роботи над проектами?

  1. Керування вихідним кодом.
  2. Хмарний графічний інтерфейс.
  3. Спільна робота над проектами, керування проектами, відстеження помилок.
  4. Система віддаленого доступу до локальних репозиторіїв.

10 Які етапи активації та роботи з Git у проекті правильно відображені в наведеному описі?

  1. Ініціалізація Git у проекті та перевірка стану файлів.
  2. Налаштування ігнорування файлів та подальша перевірка стану файлів.
  3. Подив на історію комітів та перевірка стану файлів після коміту.
  4. Індексація файлів для наступного коміту та здійснення першого коміту.

Знайомство із CSS. Підключення CSS.

1 Як вбудовані стилі CSS задаються в HTML?

  1. За допомогою тега < link>.
  2. В атрибуті style HTML-тега.
  3. За допомогою тега < style>.
  4. В окремому файлі з розширенням .css.

2 Які є переваги використання внутрішніх стилів CSS?

  1. Легко впроваджувати стилі на окремі сторінки HTML
  2. Зберігають структуру HTML і CSS разом в одному документі.
  3. Дозволяють легко застосовувати одні й ті ж стилі до багатьох сторінок.
  4. Усі відповіді вірні.

3 Яким чином підключаються зовнішні стилі CSS до HTML-документа?

  1. За допомогою тега < style>.
  2. Через вбудовані стилі в HTML-тегах.
  3. За допомогою тега < link>.
  4. Все відповіді невірні.

4 Які є переваги використання зовнішніх стилів CSS?

  1. Дозволяють змінювати стилі для окремих елементів.
  2. Зберігають структуру та чистоту HTML-коду.
  3. Потрібно підтримувати окремий файл для стилів.
  4. Ускладнюють підтримку та оновлення стилів для багатьох сторінок.

5 Що означає термін "Каскадність" в контексті CSS?

  1. Можливість створення анімаційних ефектів на веб-сайтах.
  2. Алгоритм, що визначає, як поєднувати та суміщувати значення властивостей з різних джерел CSS-правил.
  3. Застосування шрифтів для поліпшення читабельності тексту.
  4. Спосіб організації медіа-запитів для адаптивності веб-дизайну.

6 Що таке "Рідні стилі браузера" (Browser Default Styles) в контексті веб-розробки?

  1. Набір стилів, які визначаються власником веб-сайту для стандартного відображення контенту.
  2. Стилі, які браузер автоматично застосовує до елементів HTML, якщо вони не мають власних стилів.
  3. Алгоритм вибору шрифтів для текстового вмісту на веб-сайтах.
  4. Параметри стилізації, які визначають розташування елементів на веб-сторінці.

7 Який синтаксис використовується для вибору елементів на веб-сторінці за допомогою селекторів CSS?

  1. Селектори визначаються за допомогою тегів HTML.
  2. Селектори включають в себе лише класи та ідентифікатори.

1 Які основні властивості анімації в CSS і для чого вони використовуються?

  1. animation-style, animation-duration, animation-reverse - для задання стилю, тривалості та напрямку анімації.
  2. animation-name, animation-delay, animation-fill-mode - для визначення імені, затримки та режиму заповнення анімації.
  3. animation-type, animation-length, animation-direction - для визначення типу, тривалості та напрямку анімації.
  4. animation-name, animation-duration, animation-direction - для визначення імені, тривалості та напрямку анімації.

2 Які варіанти значень може приймати властивість animation-direction?

  1. normal, reverse, alternate, both
  2. up, down, left, right
  3. start, end, middle
  4. forward, backward, loop, none

3 Яким чином визначаються ключові кадри (keyframes) в CSS для анімації?

  1. Вони визначаються за допомогою тегів < keyframe> в CSS файлі.
  2. Вони визначаються за допомогою властивостей keyframe-start та keyframe-end в CSS.
  3. Ключові кадри визначаються за допомогою ключових слів from та to, або відсотків від 0% до 100%.
  4. Ключові кадри автоматично визначаються для кожної анімації без спеціальної декларації.

4 Як працює прелоадер у веб-розробці та чому він використовується?

  1. Прелоадер використовується для завантаження веб-сторінок миттєво та уникнення будь-якого затримання для користувачів.
  2. Прелоадер показується, коли веб-сторінка або програма завантажується, щоб вказати користувачеві, що процес триває, та запобігти враженню, що сторінка зламалася.
  3. Прелоадер використовується для забезпечення безпеки користувачів під час завантаження сторінок.
  4. Прелоадер показується лише тоді, коли на сторінці є велика кількість важливого контенту.

5 Які CSS властивості та анімації використовуються для створення прелоадера у наведеному коді?

  1. preloader::before, border-radius, @keyframes rotating - для створення кругового відображення.
  2. preloader::after, border, @keyframes rotating - для створення кругового відображення.
  3. preloader::before, box-sizing, @keyframes rotating - для створення кругового відображення.
  4. preloader::after, transform, @keyframes rotating - для створення кругового відображення.

6 Що таке SVG анімація та в чому полягає її перевага у порівнянні із звичайними зображеннями?

  1. SVG анімація використовує векторну графіку для створення анімованих ефектів; її перевага полягає в тому, що зображення можна масштабувати без втрати якості.
  2. SVG анімація - це спосіб анімувати зображення в Інтернеті, і її перевага полягає у використанні піксельних даних.
  3. SVG анімація використовується лише для великих графічних проектів.
  4. SVG анімація полягає в створенні статичних зображень без анімаційних ефектів.

7 Коли варто використовувати SVG анімації згідно з текстом?

  1. Анімації SVG варто використовувати лише для складних графічних проектів.
  2. Анімації SVG найкраще підходять для анімації елементів, що повторюються, наприклад, логотипу.
  3. Анімації SVG варто використовувати тільки в поєднанні з піксельною графікою.
  4. Анімації SVG найефективніше використовувати для анімації текстового контенту.

8 Як можна контролювати анімацію SVG за допомогою властивостей CSS?

  1. Зміною розміру зображення SVG.
  2. Використання властивостей, таких як animation-delay, animation-duration, animation-iteration-count та animation-timing-function.
  3. Лише за допомогою спеціальних програм для редагування SVG.
  4. Застосування тільки властивостей трансформації.

9 Які два моменти згадує автор щодо стилізації чекбоксів у початковому описі?

  3. input повинен бути всередині label, щоб змінювати свій стан при кліці на label.
  4. Усі варіанти відповідей вірні.

10 Як автор використовує псевдоелемент ::before для створення кастомного чекбокса в останньому прикладі?

  4. Застосовує псевдоелемент ::before до input для приховання його.
